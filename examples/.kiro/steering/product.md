# Product Overview

## 사내 회의실 예약 시스템

회사 내 모든 구성원이 실시간으로 회의실 사용 현황을 파악하고 손쉽게 예약할 수 있는 웹 기반 시스템입니다. 수동 예약으로 인한 중복 예약과 예약 누락을 방지하고, 회의실 자원을 효율적으로 관리하여 구성원들이 본연의 업무에 집중할 수 있도록 지원합니다.

## Core Features

### 1. 회의실 사용현황 대시보드
- 모든 회의실의 예약 상태를 한눈에 확인
- 시간대별 예약 현황을 시각적으로 표시 (30분 단위)
- 회의실 정보(이름, 수용 인원, 시설) 표시
- 실시간 예약 상태 업데이트
- 운영 시간(08:00~20:00) 내 시간 슬롯 표시
- 예약 가능/불가능 상태 시각적 구분

### 2. 날짜 선택 및 탐색
- 직관적인 날짜 선택 인터페이스
- 오늘부터 30일 이내 날짜 선택 가능
- 선택한 날짜의 예약 현황 즉시 표시
- 범위 외 날짜 선택 시 안내 메시지

### 3. 회의실 예약 관리
- 빈 시간대 클릭으로 간편한 예약 생성
- 예약 정보 입력 (목적 등)
- 예약 시간 검증 (30분 단위, 최소 30분~최대 2시간)
- 자신의 예약 수정 및 취소
- 예약 생성/수정 시 즉시 대시보드 반영
- 예약 삭제 시 확인 대화상자

### 4. 관리자 회의실 관리
- 새로운 회의실 등록
- 회의실 정보 수정 (이름, 수용 인원, 시설)
- 회의실 삭제 (활성 예약 없는 경우)
- 회의실 이름 중복 방지
- 삭제 시 활성 예약 확인 및 경고

### 5. 관리자 예약 관리
- 모든 사용자의 예약 조회
- 예약자 정보 포함 상세 정보 표시
- 모든 예약 수정 및 삭제 권한
- 긴급 상황 대응 가능

### 6. 인증 및 권한 관리
- OAuth2 기반 로그인
- 사용자 역할 자동 식별 (일반 사용자/관리자)
- 역할별 기능 접근 제어
- 예약 소유권 검증

### 7. 반응형 디자인
- 모바일 기기 최적화 인터페이스
- 데스크톱 최적화 레이아웃
- 화면 크기 자동 조정
- 모든 핵심 기능 모바일 접근 가능

## Business Rules

### 예약 시간 규칙
- **시간 단위**: 예약은 30분 단위로만 가능
- **최소 시간**: 최소 30분 이상 예약 필요
- **최대 시간**: 최대 2시간까지 예약 가능
- **운영 시간**: 08:00~20:00 범위 내에서만 예약 가능
- **시간대 정렬**: 시작 시간과 종료 시간이 30분 단위로 정렬되어야 함

### 예약 기간 규칙
- **예약 가능 기간**: 오늘부터 최대 30일 이내만 예약 가능
- **과거 날짜**: 과거 날짜는 예약 불가
- **시간대 기준**: 모든 시간은 UTC+9 (한국 표준시) 기준

### 예약 제한 규칙
- **사용자당 예약 제한**: 한 사용자당 최대 3개의 활성 예약 보유 가능
- **활성 예약 정의**: 현재 시간 이후의 예약만 활성 예약으로 계산
- **제한 초과 시**: 새로운 예약 생성 거부 및 안내 메시지 표시

### 예약 충돌 방지 규칙
- **시간대 중복 불가**: 동일한 회의실의 같은 시간대에 중복 예약 불가
- **충돌 감지**: 시작 시간 또는 종료 시간이 겹치는 모든 경우 감지
- **충돌 시 거부**: 충돌 발생 시 예약 생성/수정 거부 및 안내 메시지

### 회의실 관리 규칙
- **이름 유일성**: 회의실 이름은 중복될 수 없음
- **필수 정보**: 이름, 수용 인원, 시설 정보 필수 입력
- **삭제 제약**: 미래의 활성 예약이 있는 회의실은 삭제 불가
- **삭제 경고**: 삭제 불가 시 남아있는 예약 개수 표시

### 권한 관리 규칙
- **인증 필수**: 모든 기능 사용 시 인증 필요
- **예약 소유권**: 자신의 예약만 수정/삭제 가능 (관리자 제외)
- **관리자 권한**: 관리자는 모든 회의실 및 예약 관리 가능
- **일반 사용자 제한**: 일반 사용자는 회의실 관리 및 타인 예약 관리 불가

### 데이터 일관성 규칙
- **실시간 반영**: 예약 생성/수정/삭제 시 대시보드 즉시 업데이트
- **트랜잭션 보장**: 모든 데이터 변경 작업은 트랜잭션으로 원자성 보장
- **롤백 처리**: 오류 발생 시 모든 변경 사항 자동 롤백

## Target Users

- **일반 사용자 (User)**: 회사 내 모든 구성원으로, 회의실 예약 현황을 조회하고 자신의 예약을 생성, 수정, 취소할 수 있습니다. 최대 3개의 활성 예약을 보유할 수 있으며, 다른 사용자의 예약이나 회의실 정보는 관리할 수 없습니다.

- **관리자 (Administrator)**: 시스템 관리 권한을 가진 사용자로, 모든 사용자의 예약을 조회 및 관리할 수 있고, 회의실 정보를 등록, 수정, 삭제할 수 있습니다. 긴급 상황이나 관리 목적으로 모든 예약을 조정할 수 있는 전체 권한을 보유합니다.

## User Scenarios

### 일반 사용자 시나리오

1. **로그인 및 대시보드 접근**
   - OAuth2를 통해 로그인
   - 대시보드에서 오늘 날짜의 모든 회의실 예약 현황 확인

2. **미래 날짜 예약 확인**
   - 날짜 선택 인터페이스에서 원하는 날짜 선택 (최대 30일 이내)
   - 선택한 날짜의 회의실 예약 현황 확인

3. **회의실 예약 생성**
   - 대시보드에서 비어있는 시간대 클릭
   - 예약 폼에 목적 등 정보 입력
   - 제출하여 예약 생성
   - 대시보드에서 즉시 반영된 예약 확인

4. **예약 수정**
   - 자신의 예약 선택
   - 수정 폼에서 시간 또는 정보 변경
   - 제출하여 예약 수정
   - 변경 사항 즉시 반영 확인

5. **예약 취소**
   - 자신의 예약 선택
   - 삭제 버튼 클릭
   - 확인 대화상자에서 삭제 확인
   - 대시보드에서 시간대가 예약 가능 상태로 변경 확인

6. **예약 제한 확인**
   - 3개의 활성 예약 보유 시 추가 예약 시도
   - 제한 안내 메시지 확인
   - 기존 예약 취소 후 새로운 예약 생성

### 관리자 시나리오

1. **로그인 및 전체 예약 현황 확인**
   - OAuth2를 통해 관리자로 로그인
   - 대시보드에서 모든 사용자의 예약 정보 확인
   - 각 예약의 예약자 이름 및 상세 정보 확인

2. **회의실 등록**
   - 회의실 관리 페이지 접근
   - 회의실 추가 버튼 클릭
   - 회의실 정보 입력 (이름, 수용 인원, 시설)
   - 제출하여 회의실 생성
   - 대시보드에 새 회의실 즉시 표시 확인

3. **회의실 정보 수정**
   - 회의실 관리 페이지에서 회의실 선택
   - 수정 폼에서 정보 변경 (수용 인원, 시설 등)
   - 제출하여 회의실 정보 업데이트
   - 변경 사항 반영 확인

4. **회의실 삭제**
   - 활성 예약이 없는 회의실 선택
   - 삭제 버튼 클릭
   - 확인 대화상자에서 삭제 확인
   - 대시보드에서 회의실 제거 확인

5. **회의실 삭제 제약 확인**
   - 활성 예약이 있는 회의실 삭제 시도
   - 경고 메시지 및 남아있는 예약 개수 확인
   - 삭제 거부 확인

6. **타 사용자 예약 관리**
   - 대시보드에서 임의의 사용자 예약 선택
   - 예약 수정 또는 삭제 수행
   - 긴급 상황 대응 또는 관리 목적으로 예약 조정

7. **시스템 모니터링**
   - 전체 회의실 사용률 확인
   - 예약 패턴 분석
   - 필요시 회의실 추가 또는 제거 결정

## Key Metrics

### 사용자 경험 지표
- **예약 소요 시간**: 평균 1분 이내 예약 완료
- **예약 충돌률**: 0% (시스템 자동 방지)
- **예약 취소율**: 추적 및 분석
- **모바일 사용률**: 전체 사용의 40% 이상

### 시스템 성능 지표
- **페이지 로딩 시간**: 2초 이내
- **API 응답 시간**: 500ms 이내
- **동시 사용자**: 최소 100명 지원
- **가용성**: 99.9% 이상

### 비즈니스 지표
- **회의실 사용률**: 평균 70% 이상
- **예약 정확도**: 99% 이상 (누락/중복 최소화)
- **사용자 만족도**: 설문조사 기반 측정
- **시간 절감**: 수동 예약 대비 80% 시간 절감

## Success Criteria

1. **기능 완성도**: 모든 요구사항 구현 및 테스트 완료
2. **안정성**: 프로덕션 환경에서 오류 없이 동작
3. **사용성**: 사용자 교육 없이 직관적 사용 가능
4. **성능**: 응답 시간 및 로딩 시간 목표 달성
5. **보안**: 인증/권한 관리 정상 작동
6. **확장성**: 사용자 증가에 따른 자동 확장 지원
7. **유지보수성**: 코드 품질 및 테스트 커버리지 목표 달성
