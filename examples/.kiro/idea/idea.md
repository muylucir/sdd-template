## 1️⃣ 아이디어 소개

### 한 문장 요약
```
"사내 회의실 예약 시스템 관리도구"


```

### 배경 (왜 이 아이디어인가?)
```
회의실 예약을 수동으로 하고 있어서 중복되거나 누락되는 경우가 너무 많습니다.
회사내 구성원 모두가 공통적으로 회의실 사용현황을 파악하고 예약할 수 있는 시스템을 만들려고 합니다.
웹 기반으로 접근성을 높이고 모바일에서도 사용할 수 있도록 반응형 디자인을 적용할겁니다.


```

### 핵심 가치
```
회사내 구성원들이 손쉽게 회의실을 예약하고 사용할 수 있습니다.
회의실 예약에 낭비되는 시간을 줄이고 업무에 집중할 수 있습니다.


```

---

## 2️⃣ 구현 아이디어

### 기본 콘셉트
```
기본적으로 대시보드에서 회의실별 예약 현황을 파악할 수 있고 필요하면 원하는 날짜로 이동해서 시간대를 클릭하고 예약/수정/예약취소 할 수 있습니다.

```

### 필요한 주요 기능
```
- Feature 1: 회의실 사용현황 대시보드. 대시보드에는 날짜를 이동할 수 있는 인터페이스 제공
- Feature 2: 대시보드에서 원하는 날짜, 원하는 시간에 회의실 예약/수정/취소 기능. 회의실은 30분단위로 예약 가능. 
- Feature 3: 관리자와 일반사용자를 나누는 권한관리 체계. 관리자는 모든 사양자의 예약을 보고 관리할 수 있음
- Feature 4: 관리자는 회의실 등록/수정/삭제 가능


```

### 사용자와 시나리오
```
누가 사용하나요?: 회사내 모든 구성원이 사용합니다.


사용 시나리오:
사용자가 대시보드를 보고 비어있는 회의실을 예약할 수 있습니다.
회의실은 최소 30분, 최대 2시간까지 예약 가능.
관리자는 회의실 규모, 시설, 사용가능 시간을 등록/수정/삭제 할 수 있습니다.
회의실을 오늘 부터 최대 한달 이내의 기간에만 예약할 수 있습니다.
한 사람당 최대 3개의 회의실가지만 예약 할 수 있습니다.
회의실 예약 가능시간은 08:00~20:00 까지 입니다.


```

## 3️⃣ 기술 & 실현성

### 기술 스택 (생각하고 있는 것)
```
언어/프레임워크: 프론트엔드와 백엔드 모두 NextJS15사용. UI는 Shadcn 사용 UI 컴포넌트는 직접 코드 작성하지말고 반드시 shadcn의 컴포넌트를 설치해서 사용

데이터베이스: Aurora MySQL Serverless v2

기타 주요 기술: AWS Amplify를 사용해 배포합니다. 인증인가는 oauth2를 사용, 타임존은 UTC+9이 기준

```
